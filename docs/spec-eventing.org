* Overview

An event-driven framework for tracking, reporting, and analyzing bot behavior
on sites using the =ao= library. Provides server-side classification hooks,
client-side behavioral verification (for JS-capable agents), and an analytics
pipeline for secondary analysis.

* Architecture

* Anonymous Session Tracking

** Session ID generation

No PII stored. Session ID is a one-way hash that allows correlating
server-side and client-side events for the same visit.

#+begin_src javascript
import { createHash } from 'crypto';

function generateSessionId(ip, userAgent, date) {
  const daily = date.toISOString().slice(0, 10); // rotate daily
  return createHash('sha256')
    .update(`${ip}:${userAgent}:${daily}`)
    .digest('hex')
    .slice(0, 16); // 16-char hex = 64 bits entropy
}
#+end_src

** Properties

- Deterministic: same visitor → same ID within a day
- Non-reversible: cannot recover IP or UA from hash
- Rotates daily: no long-term tracking
- Correlates server + client events for same request

** Beacon behavior (ao.js — ~1KB)

#+begin_src javascript
(function() {
  const endpoint = document.currentScript.dataset.endpoint || '/ao/beacon';
  const sid = document.cookie.match(/ao_sid=([^;]+)/)?.[1] || '';

  // Phase 1: page_view (fires immediately)
  beacon({ type: 'page_view', sid });

  // Phase 2: behavior signals (fires after interaction)
  const signals = new Set();
  const handlers = {
    scroll: () => signals.add('scroll'),
    mousemove: () => signals.add('mouse'),
    touchstart: () => signals.add('touch'),
    focus: () => signals.add('focus'),
    keydown: () => signals.add('keyboard'),
  };

  for (const [event, handler] of Object.entries(handlers)) {
    document.addEventListener(event, handler, { once: true, passive: true });
  }

  // Phase 3: behavior_proof (fires on unload or after 5s of interaction)
  let proofSent = false;
  function sendProof() {
    if (proofSent || signals.size === 0) return;
    proofSent = true;
    beacon({
      type: 'behavior_proof',
      sid,
      signals: [...signals],
      timeOnPage: Math.round(performance.now() / 1000),
    });
  }

  setTimeout(() => { if (signals.size > 0) sendProof(); }, 5000);
  document.addEventListener('visibilitychange', () => {
    if (document.visibilityState === 'hidden') sendProof();
  });

  function beacon(data) {
    const url = `${endpoint}?${new URLSearchParams(data)}`;
    if (navigator.sendBeacon) {
      navigator.sendBeacon(url);
    } else {
      new Image().src = url;
    }
  }
})();
#+end_src

** What this proves

| Signal | Human | JS Agent (Claude-User, ChatGPT-User) | Crawler (GPTBot, ClaudeBot) |
|--------|-------|--------------------------------------|----------------------------|
| =page_view= beacon fires | yes | yes (if JS enabled) | no |
| =scroll= | yes | unlikely | no |
| =mouse/touch= | yes | no | no |
| =focus= | yes | no | no |
| =timeOnPage > 5s= | yes | possible | no |
| =behavior_proof= with 3+ signals | yes | no | no |

** Labels

All metrics carry these labels:
- =category= — ai_llm, search_engines, monitoring, human, etc.
- =name= — specific bot name (GPTBot, ClaudeBot, etc.) or "unknown"
- =tier= — attestation tier 0-4
- =data_usage= — training, search, research, unknown
- =compliant= — true/false

* GTM Integration (optional)

For sites already using Google Tag Manager:

#+begin_src html
<!-- GTM custom HTML tag (fires on all pages) -->
<script>
  // Push ao classification to dataLayer
  var ua = navigator.userAgent;
  // Server sets this via response header or inline
  var aoData = window.__ao || {};
  if (aoData.isBot) {
    dataLayer.push({
      event: 'ao_bot_detected',
      ao_category: aoData.category,
      ao_name: aoData.name,
      ao_tier: aoData.tier,
      ao_data_usage: aoData.data_usage,
    });
  }
</script>
#+end_src

Note: GTM only fires for JS-capable visitors. Traditional crawlers won't
trigger GTM tags. This is useful for detecting:
- Browser automation (Puppeteer, Playwright)
- JS-rendering crawlers (Googlebot renderer, prerender.io)
- AI agents that browse with full browser (Claude-User via claude-code)

* Privacy & Legal Considerations

| Concern | Mitigation |
|---------|-----------|
| IP addresses are PII | Never store raw IP — only hash(IP+UA+date) |
| GDPR tracking consent | No cookies for bots, no cross-site tracking |
| Session correlation | Daily rotation, 16-char truncated hash |
| Data retention | Metrics aggregated, raw events TTL 7 days |
| First-party only | Beacon hits same origin, no third-party pixels |
| Bot consent | Bots have no GDPR rights — track freely |
| Human misclassification | If behavior_proof fires, don't track as bot |
