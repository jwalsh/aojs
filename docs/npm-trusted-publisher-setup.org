* Overview

Setting up OIDC-based Trusted Publisher for the =ao= npm package, linking
=github.com/jwalsh/aojs= to =npmjs.com/package/ao= via GitHub Actions.

This eliminates the need for long-lived npm tokens. Publishing is authenticated
via short-lived OIDC tokens tied to specific repository, workflow, and environment.

* Full Setup Process (Step by Step)

** Prerequisites

- npm account with 2FA enabled
- GitHub repository with Actions enabled
- Package already published at least once (to claim the name)

** Step 1: npm — Configure Trusted Publisher

#+begin_src mermaid
sequenceDiagram
    participant User
    participant NPM as npmjs.com

    User->>NPM: Navigate to package Settings
    User->>NPM: Trusted Publisher → GitHub Actions
    User->>NPM: Fill: org=jwalsh, repo=aojs, workflow=ci.yml, env=npm
    User->>NPM: Select allowed actions: publish, stage publish
    NPM->>User: Prompt for 2FA OTP
    User->>NPM: Enter 6-digit authenticator code
    NPM->>NPM: Save Trusted Publisher connection
#+end_src

| Field | Value | Notes |
|-------+-------+-------|
| Publisher | GitHub Actions | Only option currently |
| Organization/user | jwalsh | GitHub username or org |
| Repository | aojs | Repo name (not full URL) |
| Workflow filename | ci.yml | Must exist in =.github/workflows/= |
| Environment name | npm | Must match GH environment exactly |
| Allowed actions | publish, stage publish | At least one required |

** Step 2: npm — Set Publishing Access

| Option | Use Case |
|--------+----------|
| Require 2FA or granular token | During testing (allows local =npm publish --otp=) |
| Require 2FA and disallow tokens | Production (CI-only publishing via OIDC) |

Start with the permissive option. Tighten after verifying OIDC works.

** Step 3: GitHub — Create Environment

1. Repository → Settings → Environments → *New environment*
2. Name: =npm=
3. Configure:
   - Deployment branches → Selected branches → Add =main=
   - (Optional) Required reviewers for gated releases
   - (Optional) Wait timer for delay between test and publish
4. No secrets needed (OIDC replaces tokens)

** Step 4: GitHub — Configure Workflow

#+begin_src yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18, 20, 22]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      - run: npm ci
      - run: npm run build
      - run: npm test

  publish:
    needs: test
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    runs-on: ubuntu-latest
    environment: npm                    # <-- ties to GH environment
    permissions:
      contents: read
      id-token: write                   # <-- required for OIDC
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          registry-url: 'https://registry.npmjs.org'
      - run: npm ci
      - run: npm run build
      - run: npm publish --provenance --access public
#+end_src

Key points:
- =environment: npm= — must match the GH environment name AND npm config
- =id-token: write= — allows the runner to request an OIDC JWT
- =registry-url= — required by setup-node to configure =.npmrc=
- =--provenance= — attaches build attestation to the package
- No =NODE_AUTH_TOKEN= / =NPM_TOKEN= needed

** Step 5: Remove Legacy Secrets

After confirming OIDC publish works:
1. GitHub → Settings → Secrets → Delete =NPM_TOKEN=
2. npm → Publishing access → Switch to "Require 2FA and disallow tokens"

* Local Publishing Fallback

During testing, local publishing remains available (requires 2FA):

#+begin_src shell
# Build first
npm run build

# Publish with OTP from authenticator app
npm publish --access public --otp=123456
#+end_src

This works because publishing access is set to "Require 2FA or granular token".
Once OIDC is confirmed, switch to strict mode to disable this path.

* Replicating for a New Repository

To set up Trusted Publisher on a new repo/package:

#+begin_src shell
# 1. Ensure package is published at least once (claims the name)
npm publish --access public --otp=XXXXXX

# 2. Create the workflow (copy ci.yml pattern above)
mkdir -p .github/workflows
# Edit .github/workflows/ci.yml with publish job

# 3. Create GH environment via CLI (requires admin scope)
gh api repos/OWNER/REPO/environments/npm -X PUT \
  --input <(echo '{"deployment_branch_policy":{"protected_branches":false,"custom_branch_policies":true}}')

# 4. Add branch policy
gh api repos/OWNER/REPO/environments/npm/deployment-branch-policies -X POST \
  --input <(echo '{"name":"main","type":"branch"}')

# 5. Configure npm Trusted Publisher via web UI
# npmjs.com → package → Settings → Trusted Publisher → GitHub Actions
# Fill: org, repo, workflow filename, environment name

# 6. Test: bump version, push, watch CI
npm version patch
git push && git push --tags
gh run watch
#+end_src

* Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Publish job skipped | Environment doesn't exist | Create =npm= environment on GH |
| 403 from npm | Claims don't match config | Verify env name, workflow filename match exactly |
| =id-token= permission denied | Missing permissions block | Add =permissions: id-token: write= to job |
| Provenance fails | Missing =registry-url= in setup-node | Add =registry-url: 'https://registry.npmjs.org'= |
| Tests pass but publish fails | Branch not allowed in environment | Add branch to deployment branch policy |
| Local publish blocked | "Disallow tokens" enabled | Switch to permissive mode or use CI only |
